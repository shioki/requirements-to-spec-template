# 要求仕様書サンプル: SaaS の「チーム共有リンク」機能追加

## 処理イメージ図

```mermaid
flowchart LR
  adminCreate["管理者が共有リンクを発行(機能-01)"] --> tokenGenerate["トークン生成とURL返却"]
  tokenGenerate --> notifyUser["通知サービスでリンク共有(連携-01)"]
  notifyUser --> viewerAccess["外部閲覧者がURLへアクセス(利用-02)"]
  viewerAccess --> authCheck["認証/トークン有効性チェック(機能-02)"]
  authCheck --> documentView["対象ドキュメントを表示"]
  authCheck --> accessDenied["アクセス拒否を表示"]
  adminCreate --> revokeLink["管理者がリンクを失効(機能-03)"]
  revokeLink --> accessDenied
```

## 共有リンク利用シーケンス

```mermaid
sequenceDiagram
  participant Admin as ワークスペース管理者
  participant Viewer as 外部閲覧者
  participant UI as SaaS画面
  participant Api as 共有リンクAPI
  participant Auth as 認証/認可基盤
  participant Storage as ドキュメントストレージ

  Admin->>UI: 有効期限と上限回数を入力して発行依頼
  UI->>Api: 共有リンク発行リクエスト(機能-01)
  Api->>Auth: 管理者権限を検証
  Auth-->>Api: 検証OK
  Api-->>UI: 共有URLを返却
  UI-->>Admin: URLを表示

  Viewer->>UI: 共有URLへアクセス
  UI->>Api: トークン検証リクエスト(機能-02)
  Api->>Auth: トークン状態を確認
  alt トークン有効
    Api->>Storage: ドキュメント取得
    Storage-->>Api: ドキュメントデータ
    Api-->>UI: 閲覧データを返却
    UI-->>Viewer: ドキュメント表示
  else トークン無効/期限切れ
    Api-->>UI: アクセス拒否を返却
    UI-->>Viewer: エラーメッセージ表示
  end
```

## 1. プロジェクト概要

### 1.1 目的

ワークスペース外の関係者と、期限付きの閲覧専用リンクで安全に資料を共有し、問い合わせ対応の往復を減らす。

### 1.2 背景・課題

- 外部へPDFをメール添付しており、版管理とアクセス制御が困難
- 共有解除の手作業が漏れやすい

### 1.3 スコープ

**スコープ内**

- 閲覧専用リンクの発行・失効
- リンク経由の単一ドキュメント閲覧
- 監査ログへの記録

**スコープ外**

- 編集権限の付与
- オンプレミス環境向け機能

### 1.4 成功指標(KPI)

| 指標カテゴリ | 指標 | 目標値 | 測定条件 | 測定方法 |
| --- | --- | --- | --- | --- |
| 品質 | 誤共有インシデント件数 | 0件/四半期 | 本番運用 | インシデント管理 |
| 速度 | リンク発行完了時間 | 5秒以内(P95) | API(Application Programming Interface)計測 | APM(アプリ性能監視) |
| 期限 | ユーザー受け入れテスト(UAT)完了 | 2026-06-30まで | ステージング | 承認記録 |

## 5. 機能要求

| ID | 関連利用 | 機能要求 | 優先度 | 備考 |
| --- | --- | --- | --- | --- |
| 機能-01 | 利用-01 | 管理者が有効期限と閲覧回数上限を指定して共有リンクを発行したとき、システムはトークンを生成しURLを返す | 必須 |  |
| 機能-02 | 利用-02 | 閲覧者が有効なトークンでアクセスしたとき、システムは対象ドキュメントを表示する | 必須 |  |
| 機能-03 | 利用-01 | 管理者がリンクを失効させたとき、システムは以降のアクセスを拒否する | 必須 |  |

### 機能-01 受け入れ基準

- [ ] 条件: 管理者ロールでログイン済み
- [ ] 入力: 有効期限(最大30日)、閲覧回数上限(1〜100)
- [ ] 期待結果: HTTPSの共有URLが発行される
- [ ] 異常系: 期限が30日超の場合はバリデーションエラー
- [ ] 境界値: 閲覧回数上限=1で1回閲覧後は拒否

**Given-When-Then(例)**

```text
Given  管理者が対象ドキュメントの共有権限を持つ
When   有効期限を7日、閲覧上限を10回でリンク発行APIを呼ぶ
Then   システムは一意なトークンとHTTPSの共有URLを返す
```

![閲覧拒否画面。トークンが無効または期限切れであることを示すメッセージと、管理者連絡先が表示されている](../assets/screens/saas-feature/画面-02_閲覧拒否_期限切れ.png)

  - 拒否理由メッセージ(「リンクの有効期限が切れています」など)
